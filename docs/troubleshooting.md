# Troubleshooting

## 2026-04-16 connpass検索で高確率に取得失敗していた問題

### 症状
- connpass検索機能で、高確率にイベント取得へ失敗することがあった
- 特にユーザー名検索が不安定に見えた

### 原因
- `src/lib/fetchUserEvents.ts` で送信していた query parameter 名が `wstart` になっていた
- 一方で `/api/search-user` 側では `start` を受け取る実装になっていた
- そのため、取得開始位置が正しく API route に渡らず、検索結果取得が不安定になっていた

### 対応
- `src/lib/fetchUserEvents.ts` の query parameter を `wstart` から `start` に修正
- あわせて、`nickname` は `encodeURIComponent()` を使って安全に送るように修正
- `pagination` の返却も確認し、ページ送りの挙動を再確認した

### 確認結果
- ユーザー名検索：正常動作を確認
- イベントID検索：正常動作を確認
- イベントURL検索：正常動作を確認
- 以前の「高確率で取得に失敗する」状態は再現しなくなった

### 補足
- connpass のユーザー名は文字種に制限があるため、危険な文字が入りにくい設計と思われる
- 外部 API の一時的な失敗は今後もありうるため、本対応では「通常操作で安定して検索できること」を完了基準とした

## 2026-04-17 週ごとの参加数グラフで未来イベントが今週に加算される問題

### 症状
- ダッシュボードの「週ごとの参加数」グラフで、未来開催のイベントを登録すると「今週」に件数が加算されてしまった
- 例として、2026-05-26 開催イベントを登録すると、開催前にもかかわらず「今週」が 1件増えた

### 原因
- `src/app/api/dashboard-data/route.ts` の週別集計ロジックで、直近5週間の範囲外データを最新週キーへ寄せる処理が入っていた
- そのため、未来イベントも `latestWeekKey`（実質「今週」）に加算されていた

### 対応
- 週別集計で、未来日 `date > now` のイベントは集計しないように修正
- 直近5週間の範囲外データを今週へ寄せる fallback 処理を削除
- 直近5週間に入る開催済みイベントのみを加算するように整理した

### 確認結果
- 未来開催イベントを登録しても、「今週」に加算されないことを確認
- 開催済みイベントは正しい週に加算されることを確認
- イベント削除時にグラフ件数も正しく減ることを確認
- 週境界も確認済み
  - 2026-04-12（日）開催イベント → 「先週」
  - 2026-04-13（月）開催イベント → 「今週」

### 補足
- このグラフは「ログインユーザーが週ごとに何回参加ログを残したか」を表す
- 当日であっても、開催時間前であれば「今週」にカウントされない
- 現状は `events.started_at` を主な基準日として使用している
- `started_at` が無い場合のフォールバック処理（`updated_at` など）は残っているため、将来的に必要なら別途見直す
